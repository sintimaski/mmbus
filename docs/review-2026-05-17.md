### 2. Mismatches

- **NIT — `BackpressurePolicy` documented but not Python-exposed.**
  CHANGELOG [0.1.0] line 32 advertises
  `BackpressurePolicy::{Error, DropOldest}` as "selectable per-bus", but
  `PyBus::new` (src/python/bus.rs:20) only accepts
  `base_dir/capacity/slot_size/max_subscribers` — there is no
  `backpressure=` kwarg.  Python callers get the default `Error` policy
  with no way to switch to `DropOldest`.  Either expose the kwarg or
  drop the line from CHANGELOG before tagging 0.1.0.
- **NIT — doc spelling drift on `subscribe_from`.**
  `src/python/bus.rs:90` says it raises `OSError (Error::CursorTooOld)`,
  but the actual mapping in `exceptions.rs:33` raises `CursorTooOldError`
  (a `PyException` subclass, not `OSError`).  The Python wrapper
  docstring (`__init__.py:96`) is correct; only the Rust-side docstring
  is stale.

### 3. Exception type round-trip

Every typed exception in `__all__` has both a `create_exception!`
declaration and a matching raise site in `mmbus_err`, and every Rust
`Error` variant has at least one constructor in core code.  No orphan
types.  **OK.**

### 4. Surprises

- **NIT — `Subscription.__module__ == "builtins"` and
  `TopicStats.__module__ == "builtins"`.** PyO3 `#[pyclass(name =
  "Subscription")]` without a `module = "mmbus._mmbus"` attribute
  leaves `__module__` set to `builtins`.  Cosmetic but affects
  `pickle`, `repr`, and Sphinx cross-refs.  `AsyncSubscription`/`Bus`/
  `BusFullError` correctly show `mmbus`.
- **OK — `_RustBus` is intentionally exposed (so power users can
  subclass) but correctly excluded from `__all__`.**
- **OK — `AnyioSubscription` lazy-imports `anyio` in `__init__`,**
  matching the CHANGELOG's "opt-in" claim.

### Verdict

Ship-able for 0.1.0.  Two NITs worth fixing in the same release window:
drop or wire up the `BackpressurePolicy` CHANGELOG line, and set
`module = "mmbus._mmbus"` on the two `#[pyclass]` decorators so
`Subscription`/`TopicStats` stop reporting `builtins` as their module.

## Docs & release loose ends

### 1. OWNER/REPO placeholders — **BLOCKER**

Five literal `OWNER` references must be plugged in before tagging:
- `README.md:5-7` — comment + rustdoc URL `https://OWNER.github.io/mmbus/`
- `CHANGELOG.md:126-127` — `[Unreleased]` and `[0.1.0]` compare/tag URLs
  (these are malformed too: `https://OWNER/mmbus/...` is missing
  `github.com/`, so even after substitution they need `github.com/`
  inserted)
- `SECURITY.md:50` — security-advisory URL `https://github.com/OWNER/mmbus/...`

### 2. Broken internal links — **OK**

All 13 relative `](path)` links in `README.md`/`CHANGELOG.md`/
`SECURITY.md`/`docs/*.md` resolve.

### 3. `__version__` agreement — **OK**

`Cargo.toml:3`, `pyproject.toml:7`, and `python/mmbus/__init__.py:358`
all pin `0.1.0`.

### 4. Wire format v4 — **NIT**

Code is canonically v4 (`src/ring.rs:46` MAGIC ends in `0004`,
`OFF_VERSION` writes `4` at line 135).  `CHANGELOG.md:40,44,124`
correctly says v4.  But `docs/architecture.md` never mentions v4 and
its "Crash Safety" section describes a `dirty` flag mechanism that does
not exist in the codebase — the actual design uses a per-slot seqlock +
header generation counter.  The "Ring Buffer Design" diagram also omits
the seq field and the per-subscriber cursor table.  Architecture doc is
stale relative to shipped code.

### 5. Roadmap accuracy — **NIT**

Mostly accurate, but two drift points vs `CHANGELOG.md`:
- `roadmap.md:95-96` marks "Fuzz testing of ring buffer" as
  **unchecked** and "out of scope for v0.1", yet CHANGELOG ships the
  `fuzz/` harness with two targets (and `.github/workflows/fuzz.yml`
  runs them).  Should be `[x]`.
- `roadmap.md:18` says crash-safe restart uses "v3 wire format";
  CHANGELOG and code say v4.

### 6. Examples runnable — **OK**

`examples/fastapi_broadcast.py` (85 lines) and `examples/np_pipeline.py`
(128 lines) both end cleanly, imports look complete.

### 7. Docker — **OK**

`Dockerfile` (multi-stage builder→test) and `docker-compose.yml` exist.
The `test` service builds the `test` target.

### 8. CI workflows — **OK**

All four (`ci.yml`, `wheels.yml`, `docs.yml`, `fuzz.yml`) present and
on current action versions (Swatinem/rust-cache@v2 across the board
after N7).

### 9. LICENSE file — **BLOCKER**

`ls /Users/dd/quests/mmbus/LICENSE*` returns no matches.  README:284
says "MIT" and `Cargo.toml` lacks a `license = "MIT"` field as well.
PyPI/crates.io publish will warn or fail without metadata; a `LICENSE`
file is also required for the MIT claim to be legally enforceable.

### 10. Perf table reproducibility — **OK**

`Cargo.toml:30-36` declares `[[bench]] name = "ring"` and `[[bench]]
name = "e2e"`; `benches/compare.py` exists.  All three commands in
README:180-181 are runnable.

### Verdict

**Two blockers** before tagging v0.1.0: (a) replace every `OWNER`
placeholder (and fix the malformed `https://OWNER/mmbus/...` URLs in
CHANGELOG to include `github.com/`); (b) add a `LICENSE` file plus
`license = "MIT"` in `Cargo.toml`.  **Two nits**: refresh
`docs/architecture.md` to describe the v4 seqlock + generation counter
(drop the obsolete `dirty` flag wording) and update `docs/roadmap.md`
(mark fuzz testing shipped; correct "v3" → "v4").  Everything else is
release-ready.
